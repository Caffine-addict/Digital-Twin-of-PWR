# Digital Twin of a Nuclear Reactor Cooling System with Predictive Analytics and Anomaly Detection

## ABSTRACT

This paper presents a master's-level enhanced digital twin of a Pressurized Water Reactor (PWR) cooling loop with quantitative evaluation capabilities. The system integrates deterministic physics-based simulation with machine learning-based anomaly detection using Isolation Forest, enhanced through comprehensive evaluation metrics. Real-world dataset integration includes the NPPAD (Nuclear Power Plant Accident Dataset) containing actual PWR transient data from various accident scenarios. The evaluation framework comprises quantitative metrics including detection rate, false positive rate, detection delay, and prediction mean absolute error. System performance is validated across five predefined fault scenarios including normal operation, pump failure, overheating, pressure spike, and combined fault conditions. All features are optional and togglable via a Streamlit dashboard while maintaining backward compatibility with existing core simulation logic. Results demonstrate effective fault detection for pump failure conditions (78.4% detection rate) with quantitative validation across multiple scenarios. The framework enables reproducible research and hypothesis testing for nuclear cooling system anomaly detection.

**Keywords**—Digital Twin, PWR Cooling Loop, Anomaly Detection, Isolation Forest, NPPAD Dataset, Evaluation Metrics, Scenario Analysis, Nuclear Systems

---

## 1. INTRODUCTION

Digital twin technology has emerged as a transformative approach for nuclear power plant monitoring and predictive maintenance. By creating virtual replicas of physical systems, operators can simulate operational scenarios, predict potential failures, and optimize maintenance schedules without risking actual reactor operation. In nuclear applications, digital twins provide critical safety margins through early anomaly detection and failure prediction.

This work presents an enhanced PWR cooling loop digital twin with comprehensive evaluation capabilities that elevate the system from a basic simulation tool to a research-grade validation platform. The primary contribution is the addition of formal evaluation metrics and scenario-based testing that enable quantitative assessment of anomaly detection algorithms.

---

## 2. RELATED WORK

Digital twin implementations for nuclear applications have evolved from basic physics simulators to integrated systems combining machine learning for anomaly detection. Existing approaches include physics-based reactor simulators, data-driven anomaly detection systems, and hybrid physics-machine learning frameworks. However, comprehensive evaluation frameworks with standardized metrics remain limited in open-source implementations. The NPPAD dataset provides real PWR transient data enabling validation against actual accident scenarios.

---

## 3. SYSTEM OVERVIEW

The PWR digital twin system follows an Analysis-Numerics-Testing (A.N.T) architecture. The system integrates deterministic physics simulation with machine learning-based anomaly detection. The pipeline consists of: (1) synthetic or real data generation, (2) fault injection per schedule, (3) deterministic physics simulation, (4) feature engineering, (5) Isolation Forest model training and scoring, (6) optional evaluation modules, and (7) Streamlit dashboard for visualization. Core simulation logic remains untouched ensuring backward compatibility.

---

## 4. MATHEMATICAL MODELING

The PWR cooling loop dynamics follow locked mathematical equations:

**Temperature Dynamics:**
T_{t+1} = T_t + α(Q_in - k_c F_t (T_t - T_amb))

**Flow Relationship:**
F_t = F_base × η_pump × S_pump

**Pressure Relationship:**
P_t = k_p × F_t × T_t

**Sensor Measurement:**
X_measured = X_true + N(0,σ)

All equations implement clamping to physical domains before noise addition to ensure realistic bounds.

---

## 5. DATA GENERATION AND REAL DATA INTEGRATION

Data generation includes synthetic inputs following normal operational profiles and real-world dataset integration via the NPPAD dataset. The NPPAD dataset contains 2,434 MDB files with 69 sensors per timestep from various accident scenarios including ATWS, FLB, LOCA, and normal operation. The system converts MDB files to CSV format with columns: time, temperature, pressure, flow_rate. When real_csv data source is selected in the dashboard, the system loads and scores actual NPPAD transient data.

---

## 6. FAULT MODELING

Three primary fault types are implemented:

- **Pump Failure:** effective_η = η_PUMP × (1 - magnitude)
- **Overheating:** Q_in_effective = Q_in × (1 + magnitude)
- **Pressure Spike:** k_p_effective = k_P × (1 + magnitude)

Faults are applied via schedule specifying fault_type, start_time, end_time, and magnitude. Multiple faults can be active simultaneously.

---

## 7. FEATURE ENGINEERING

Feature engineering creates a 10-step sliding window dataset with statistical features for each sensor variable (temperature, pressure, flow_rate): mean, standard deviation, min, max, range, first difference, and second difference. This yields 21 features per window providing temporal context essential for detecting anomalous patterns.

---

## 8. MACHINE LEARNING APPROACH

The anomaly detection employs Isolation Forest trained on normal operation data. The algorithm isolates anomalies through random partitioning where anomalies require fewer splits. Anomaly scores map to status levels: NORMAL (≥ -0.1), WARNING (-0.2 to -0.1), CRITICAL (< -0.2). Optional LSTM and linear regression enable multi-horizon forecasting.

---

## 9. EVALUATION METRICS

Four quantitative metrics assess system performance:

- **Detection Rate (DR):** Fraction of fault samples correctly identified
- **False Positive Rate (FPR):** Fraction of normal samples incorrectly flagged
- **Detection Delay (DD):** Average time steps from fault start to first detection
- **Prediction MAE:** Mean absolute error between predicted and actual sensor values

---

## 10. EXPERIMENTAL SETUP

Experimental configuration uses SEED=42 for reproducibility. Five predefined evaluation scenarios include: normal (no faults), pump_failure (200-800 timesteps), overheating (200-800 timesteps), pressure_spike (200-800 timesteps), and combined_faults (multiple overlapping faults).

---

## 12. DISCUSSION

Results demonstrate effective detection for pump failure conditions (78.4% detection rate) and combined fault scenarios (41.8% detection rate). The model achieved minimal false positives during controlled scenario testing (FPR≤0.065). Detection delay was zero across all scenarios indicating immediate fault detection when anomalies were identified. The zero detection rate for overheating and pressure spike faults suggests these patterns may require enhanced feature engineering or fault-specific detection approaches.

---

## 13. LIMITATIONS

The evaluation is based on limited scenario sets and specific fault magnitudes. High false positive counts on real NPPAD data suggest potential domain shift between synthetic training and real-world data. Detection delay measurement may not capture incipient faults. Computational performance metrics were not evaluated.

---

## 14. FUTURE WORK

Future enhancements include expanding fault scenario libraries, implementing fault-specific detection models, extending NPPAD integration with fault labeling, incorporating uncertainty quantification, and developing adaptive threshold optimization.

---

## 15. CONCLUSION

The PWR digital twin system was successfully enhanced with comprehensive evaluation capabilities including quantitative metrics, method comparison tables, scenario analysis, and error analysis. All additions maintain backward compatibility while enabling research-grade validation. Results demonstrate effective fault detection for specific failure modes while highlighting areas for improvement.
