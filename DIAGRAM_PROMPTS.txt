## DIAGRAM 5: EVALUATION METRICS COMPUTATION

**Title:** Quantitative Evaluation Metrics

**Description:**
Create a 2x2 grid showing metric computation:

- Top-left box "False Positive Rate (FPR)":
  - Formula: FP / (FP + TN)
  - Visual: Highlight normal period with false alarm markers in red
  
- Top-right box "Detection Rate (DR)":
  - Formula: TP / (TP + FN)  
  - Visual: Highlight fault period with detection markers in green
  
- Bottom-left box "Detection Delay (DD)":
  - Formula: Average(time_from_fault_start_to_detection)
  - Visual: Timeline showing fault interval and first detection point
  
- Bottom-right box "Prediction MAE":
  - Formula: Mean(|actual - predicted|)
  - Visual: Two overlapping lines (actual vs predicted)

- Arrow from each box showing computation input: DataFrame with anomaly_flag, fault_class columns
- Colors: Use standard metric colors (FPR=red, DR=green, DD=blue, MAE=purple)
- Include title "Figure 5: Evaluation Metrics Computation"

# USAGE INSTRUCTIONS

1. Create diagrams one at a time using the prompts above
2. Maintain consistent styling across all diagrams:
   - Professional color palette (blues, greens, oranges)
   - Sans-serif fonts throughout
   - Consistent line weights
3. Add figure numbers and captions in each diagram
4. Reference diagrams in your report text as "Figure X shows..."
5. Export as PNG or SVG for high-quality printing

The diagrams should be placed in your report as:
- Figure 1 (after "System Overview" section)
